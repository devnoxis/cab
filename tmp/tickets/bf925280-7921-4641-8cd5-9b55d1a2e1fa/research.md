# Research: Domain Name Options for Autobot

## Problem Understanding

The ticket asks for:
1. **6 domain name options** for the Autobot project
2. **2 URLs to similar projects** in the same space

## Relevant Context and Background

**Project:** Autobot is a Ruby/Rack-based web API serving as the backend for an AI workflow automation company. Per the README and `app.rb`, Autobot is described as:

> "A company creating advanced AI workflows that help with building digital products."

The `/about` page content reinforces this positioning — the product is aimed at development teams who want to integrate AI into their digital product workflows (coding, code review, testing, deployment).

**Brand identity signals from the codebase:**
- Name: **Autobot**
- Tagline: *"AutoBot AI workflow is near"*
- Value props: Accelerated Development, Intelligent Code Review, Smart Testing & QA, Automated Documentation, Deployment Automation, Continuous Optimization
- Target audience: Engineering teams / digital product teams
- Positioning: AI as a co-creator in software development

## Domain Name Options

The following 6 domain names align with the Autobot brand, the AI workflow niche, and the digital product development audience:

| # | Domain | Rationale |
|---|--------|-----------|
| 1 | **autobot.ai** | Exact brand match with a premium AI TLD. Most direct and memorable. |
| 2 | **getautobot.com** | Classic SaaS pattern (`get<product>.com`). Friendly, action-oriented. |
| 3 | **autobotai.com** | Brand + AI suffix on a .com. Safe fallback if `autobot.ai` is taken. |
| 4 | **buildwithautobot.com** | Highlights the product's value ("build with AI"). Appeals to developers. |
| 5 | **autobot.dev** | Targets developer audience directly; `.dev` TLD signals a dev-first product. |
| 6 | **flowbot.ai** | Alternative brand direction emphasizing "AI workflow automation" if `autobot.*` is unavailable. |

## Similar Projects (URLs)

1. **n8n** — https://n8n.io/
   Open-source AI workflow automation platform with a visual builder and code-level control. Very similar positioning: integrates AI into developer workflows, supports self-hosting, and targets technical teams building automated pipelines.

2. **Dify** — https://dify.ai/
   Leading agentic workflow builder that democratizes AI agent development. Combines no-code visual flows with LLM-powered automation for digital product teams — closely mirrors Autobot's stated mission.

## Technical Considerations

- The `.ai` TLD is highly competitive and often expensive (Anguilla registry); pricing can be $50–100+/year.
- The `.dev` TLD (Google registry) requires HTTPS by default — already a best practice for any production API.
- `autobot` is also the name of a Transformers character and a GitHub bot tool, so trademark searches should be conducted before committing to any domain.

## Open Questions

- Is `autobot.ai` already taken or reserved? (Likely — it's a premium term.)
- Does the team want to keep the "Autobot" name long-term, or is this a working title?
- Should the domain reflect the API/developer product specifically, or the broader company brand?
- Is there a budget constraint for premium TLDs (`.ai`, `.io`)?
